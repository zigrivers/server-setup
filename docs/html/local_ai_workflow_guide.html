Antigravity CLI Workflow

Unified Local AI Stack Workflow Guide

An interactive guide for developers and non-technical users to build features, write tests, and run tasks using your local Apple Silicon split-machine AI models.

01

How delegation works

The split-machine stack isolates planning and execution. A frontier model running inside your selected CLI agent acts as the architect, while local open-source models (Developer & Reviewer) running on an inference worker computer perform the heavy coding and testing in a git sandbox.

Client CLI Skills Directory MCP Registration Command Allowed-Roots Configuration
Antigravity CLI (agy) ~/.gemini/antigravity-cli/skills/ scripts/register-mcp-antigravity.sh Managed in ~/.gemini/antigravity-cli/mcp_config.json
Claude Code (claude) ~/.claude/skills/ scripts/register-mcp-claude.sh Configured internally in SQLite database via CLI arguments
Codex (codex) N/A (uses built-in planning tools) scripts/register-mcp-codex.sh Configured inside ~/.codex/config.toml
02

One-Time CLI configuration

Set up the required skills and register the MCP server bridge for your selected CLI.

Install Antigravity Skills
Copy the local stack skills (delegate-local, local-review, local-ai-status) to your user directory:
Terminal · Machine 1
cd ~/ai/local-ai-stack ./scripts/install-antigravity-skills.sh
Register the MCP Bridge with Antigravity
Wire the MCP delegate server into your Antigravity configuration file (~/.gemini/antigravity-cli/mcp_config.json):
Terminal · Machine 1
./scripts/register-mcp-antigravity.sh
Install Claude Skills
Copy the skills to your Claude Code directory:
Terminal · Machine 1
cd ~/ai/local-ai-stack ./scripts/install-claude-skills.sh
Register the MCP Bridge with Claude
Wire the MCP server into Claude Code:
Terminal · Machine 1
./scripts/register-mcp-claude.sh
Register the MCP Bridge with Codex
Register the server in Codex's internal database:
Terminal · Machine 1
cd ~/ai/local-ai-stack ./scripts/register-mcp-codex.sh
Raise Codex Timeout Settings
Open Codex's configuration file in your editor:
Terminal · Machine 1
nano ~/.codex/config.toml

Find the [mcp_servers.local-ai-delegate] section and raise the values to prevent timeouts:

toml · config.toml
startup_timeout_sec = 20 tool_timeout_sec = 3600
03

Smoke testing the connection

Before you prepare your project repositories, verify that your CLI agent can successfully speak to the local AI stack.

1. Test the MCP Server binary directly
Run the Python script directly from your terminal to verify imports and dependencies work. It should start and wait for input without error:
Terminal · Machine 1
cd ~/ai/local-ai-stack .venv/bin/python mcp/local_delegate_mcp/server.py

Press Ctrl+C to terminate the process after confirming it starts successfully (does not crash immediately with an ImportError).

2. Verify status inside Antigravity CLI
Launch agy inside the local stack directory (which is already configured as an allowed-root) and run the status skill:
Terminal · Machine 1
cd ~/ai/local-ai-stack agy

Inside the session, run:

Antigravity CLI prompt
Check the status of my local AI endpoints.
✓ Expected Output
The CLI should execute the MCP status tool and return OK for the endpoints (provided your model servers are running). If you see connection errors, ensure your models are started.
2. Verify status inside Claude Code
Launch claude inside the local stack directory and run the status skill:
Terminal · Machine 1
cd ~/ai/local-ai-stack claude

Inside the session, run:

Claude Code prompt
/skills

Select local-ai-status to ping the model endpoints.

2. Verify status inside Codex
Launch codex inside the local stack directory and call the MCP status tool manually:
Terminal · Machine 1
cd ~/ai/local-ai-stack codex

Inside the session, run the tool directly:

Codex prompt
local-ai-delegate__local_ai_status()
04

Preparing a project repository

Every project repository that you want to develop using local models needs to opt in by including local-agent instructions.

Bootstrap the Repo templates
Navigate to your target project folder (e.g. ~/ai/workspaces/my-app) and copy the instructions templates from the stack repo.
Terminal · Machine 1
cd ~/ai/workspaces/my-app STACK=~/ai/local-ai-stack mkdir -p docs/ai plans cp "$STACK/docs/repo-templates/AGENTS.md" ./AGENTS.md cp "$STACK/docs/repo-templates/CLAUDE.md" ./CLAUDE.md cp "$STACK/docs/repo-templates/AI_WORKFLOW.md" ./docs/ai/AI_WORKFLOW.md touch plans/.gitkeep git add AGENTS.md CLAUDE.md docs/ai/AI_WORKFLOW.md plans/.gitkeep git commit -m "Add hybrid AI workflow instructions"

06

Walkthrough: Build a new feature

Here is a step-by-step example of how to add a database migration and a route using your selected CLI.

1. Launch the CLI
Open the Antigravity CLI:
Terminal · Machine 1
cd ~/ai/workspaces/my-app agy
2. Propose Feature Plan
Ask agy to generate the implementation plan:
Antigravity CLI prompt
Create an implementation plan to add a users table database migration and route.
3. Delegate Execution to Local Stack
Delegate the plan execution:
Antigravity CLI prompt
/delegate-local plans/2026-05-23-add-users-db.md
1. Launch the CLI
Open Claude Code:
Terminal · Machine 1
cd ~/ai/workspaces/my-app claude
2. Propose Feature Plan
Ask Claude to create the plan:
Claude Code prompt
Create an implementation plan to add a users table database migration and route.
3. Delegate Execution to Local Stack
Delegate the task execution:
Claude Code prompt
/delegate-local plans/2026-05-23-add-users-db.md
1. Launch the CLI
Open Codex CLI:
Terminal · Machine 1
cd ~/ai/workspaces/my-app codex
2. Propose Feature Plan
Ask Codex to create the plan:
Codex CLI prompt
Create an implementation plan to add a users table database migration and route.
3. Delegate Execution to Local Stack
Codex uses tool calls directly. Run:
Codex CLI prompt
Run the local plan at plans/2026-05-23-add-users-db.md
4. Inspect & Merge
The local models will run tests, review, and write code inside the git sandbox. Once done, inspect the branch changes and merge to main:
Terminal · Machine 1
cd ~/ai/workspaces/my-app git diff ai/local/add-users-db-... # If happy, merge the local work branch git merge ai/local/add-users-db-...
07

Troubleshooting & Triage cheat sheet

A quick-reference list of common issues when executing plans, their potential causes, and how to fix them.

Symptom Common Cause Remediation Action
local_ai_status reports FAIL on an endpoint Inference server is down or Thunderbolt bridge is disconnected. Run preflight.sh to trace the network path. Restart workers via start-worker-models.sh if needed.
run_local_plan fails with Binary not found in agy The virtual environment has not been initialized or registered. Make sure you ran uv pip install -e '.[all]' on Machine 1, and ensure `LOCAL_EXEC_PLAN_BIN` in mcp_config.json points to your virtual env's bin path.
Claude Code errors during /delegate-local tool call The tool calls took longer than the default Claude Code timeout (often 5 mins). Use /audit-stack to ensure connections are responsive, or chunk the task into smaller sub-tasks.
Codex fails to execute tool or times out The tool_timeout_sec parameter inside ~/.codex/config.toml is too small. Double check that your config.toml has tool_timeout_sec = 3600.
Git worktree creation fails or collisions occur A stale worktree exists with the same branch or directory name. Run git worktree prune to clean up stale references, or remove directory manually.
