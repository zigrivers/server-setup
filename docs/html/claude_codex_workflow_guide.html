Interactive Workflow Guide

Split-Machine Development with Claude & Codex

A step-by-step guide for developers to delegate tasks, run audits, and write code using Claude Code CLI and Codex CLI connected to your local split-machine AI stack.

01

How delegation works with Claude & Codex

Just like the Antigravity setup, this workflow uses Claude Code or Codex on Machine 1 as the planning agent, which leverages the local-ai-delegate MCP bridge. When you approve a plan, the MCP bridge spawns a local execution loop that operates on your local Apple Silicon models.

Client CLI Skills Location MCP Registration Command Configuration Target
Claude Code (claude) ~/.claude/skills/ scripts/register-mcp-claude.sh Managed internally by Claude Code sqlite database
Codex (codex) N/A (uses built-in tools) scripts/register-mcp-codex.sh Statically configured in ~/.codex/config.toml
02

One-Time Claude Code CLI setup

Follow these steps to enable the three local skills and the MCP bridge for Claude Code on Machine 1.

Install Claude Skills
Copy the local-stack skills (delegate-local, local-review, local-ai-status) to Claude's global skills directory.
Terminal · Machine 1
cd ~/ai/local-ai-stack ./scripts/install-claude-skills.sh

Run /skills inside Claude Code to verify they show up.

Register the MCP Bridge with Claude Code
Wire the MCP server directly into Claude Code. This registers the server under a user scope with allowed workspace directory directories:
Terminal · Machine 1
./scripts/register-mcp-claude.sh

Type /mcp inside Claude Code and confirm local-ai-delegate is active.

03

One-Time Codex CLI setup

Follow these steps to wire up the MCP bridge and adjust timeouts for the Codex CLI.

Register the MCP Bridge with Codex
Register the server in Codex's internal database:
Terminal · Machine 1
./scripts/register-mcp-codex.sh
Edit Codex Timeout Settings
Open Codex's static configuration file in your editor:
Terminal · Machine 1
nano ~/.codex/config.toml

Find the [mcp_servers.local-ai-delegate] section and ensure timeouts are raised. Local Qwen loops take time, and standard 30-second timeouts will fail:

toml · config.toml
startup_timeout_sec = 20 tool_timeout_sec = 3600
04

Preparing a project repository

Every project repository that you want to develop using local models needs to opt in by including local-agent instructions.

Bootstrap the Repo templates
Navigate to your target project folder (e.g. ~/ai/workspaces/my-app) and copy the instructions templates from the stack repo. This defines safety guardrails and local endpoints rules.
Terminal · Machine 1
cd ~/ai/workspaces/my-app STACK=~/ai/local-ai-stack mkdir -p docs/ai plans cp "$STACK/docs/repo-templates/AGENTS.md" ./AGENTS.md cp "$STACK/docs/repo-templates/CLAUDE.md" ./CLAUDE.md cp "$STACK/docs/repo-templates/AI_WORKFLOW.md" ./docs/ai/AI_WORKFLOW.md touch plans/.gitkeep git add AGENTS.md CLAUDE.md docs/ai/AI_WORKFLOW.md plans/.gitkeep git commit -m "Add hybrid AI workflow instructions"
05

Walkthrough: Build a new feature

Here is a step-by-step example of how a developer adds a database migration and a route using Claude Code.

1. Launch Claude Code
Enter your project workspace and open claude:
Terminal · Machine 1
cd ~/ai/workspaces/my-app claude
2. Propose Feature & Generate Plan
Ask Claude to create the implementation plan:
Claude Code prompt
Create an implementation plan to add a users table database migration and route.

Claude Code will write the plan details to: plans/2026-05-23-add-users-db.md.

3. Delegate Execution to Local Stack
Once you approve the plan, delegate its execution:
Claude Code prompt
/delegate-local plans/2026-05-23-add-users-db.md
⚠️ Isolation Rule
The script will create a git worktree sandbox and checkout a new branch (e.g. ai/local/add-users-db-...) automatically so your working directory remains untouched.
4. Inspect & Merge
Exit Claude Code and verify the changes inside the worktree before merging to main:
Terminal · Machine 1
cd ~/ai/workspaces/my-app git diff ai/local/add-users-db-... # Merge the local work branch git merge ai/local/add-users-db-...
06

Troubleshooting & Triage cheat sheet

A quick-reference list of common issues when executing plans, their potential causes, and how to fix them.

Symptom Common Cause Remediation Action
local_ai_status reports FAIL on an endpoint Inference server is down or Thunderbolt bridge is disconnected. Run preflight.sh to trace the network path. Restart workers via start-worker-models.sh if needed.
Claude Code errors during /delegate-local tool call The tool calls took longer than the default Claude Code timeout (often 5 mins). Use /audit-stack to ensure connections are responsive, or chunk the task into smaller sub-tasks.
Codex fails to execute tool or times out The tool_timeout_sec parameter inside ~/.codex/config.toml is too small. Double check that your config.toml has tool_timeout_sec = 3600.
Git worktree creation fails or collisions occur A stale worktree exists with the same branch or directory name. Run git worktree prune to clean up stale references, or remove directory manually.
